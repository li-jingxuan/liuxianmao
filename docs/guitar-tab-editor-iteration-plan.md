# 六线谱编辑器迭代计划

## 1. 使用规则

- 本文档是编辑器迭代状态的唯一来源，技术方案不再重复维护详细 Milestone。
- 未经确认，不开始任何待办迭代的代码实现。
- 只有当某次迭代的功能、测试、文档和验收项全部通过后，才将状态从 `[ ]` 改为 `[x]`。
- 部分完成的迭代仍保持 `[ ]`，并在“进度记录”中注明已完成内容和阻塞项。
- 每次只推进一个迭代；需求变化先更新本文档，再开始实现。
- 当前不考虑向下兼容，数据格式变更只维护当前 `schemaVersion`。

## 3. 完成定义

一个迭代只有同时满足以下条件才算完成：

- 迭代范围内的功能已经实现，没有用静态假数据伪装交互结果。
- TypeScript 类型检查、Lint 和构建通过。
- 本迭代要求的单元测试、交互测试或视觉回归测试通过。
- 桌面端和目标最小视口完成浏览器验收，页面外层不出现非预期滚动条。
- 无阻断级错误，已知非阻断问题记录到本文档。
- 数据模型、快捷键或交互发生变化时，技术文档同步更新。
- 用户确认本迭代验收通过。

- [x] 使用 Zod 定义当前版本的完整文档 schema，并通过 `z.infer` 生成业务类型。
- [x] 实现 `JSON.parse → Zod safeParse → 语义校验` 加载管线。
- [x] 实现 ID 唯一性、跨引用、弦号、品位、时值容量和和弦引用校验。
- [x] 建立 `useScoreStore`、`useEditorStore` 和 `useViewportStore`。

### [x] Iteration 2：六线谱只读渲染与排版

目标：从真实 Score 数据稳定生成多行六线谱，不包含编辑行为。

实现范围：

- [x] 实现 `layoutScore`、`layoutSystem` 和 `layoutMeasure`。
- [x] 渲染六条弦线、小节线、小节号、拍号、速度和品位数字。
- [x] 支持休止符、多音同时发声、变拍号和跨行小节布局。
- [x] 支持附点、双附点与二至六连音编号/括号渲染。
- [x] Bravura 只用于休止符等 SMuFL 音乐符号。
- [x] 实现字体加载状态和字体加载失败提示。
- [x] 建立 layout 命中索引，为下一迭代的点击定位提供基础。
- [x] 使用固定 720p 桌面基线宽度排版，但不实现响应式和分页打印。

验收标准：

- [x] 示例数据的全部小节、音符和休止符正确渲染。
- [x] `1280 × 720` 视口下无元素重叠或文本截断。
- [x] 普通界面字体与 Bravura 不混用。
- [x] 100 小节只读谱例可流畅滚动。

测试要求：排版函数单元测试、`1280 × 720` 浏览器视觉回归、字体加载异常测试。

### [ ] Iteration 4：选区、剪贴板与撤销重做

目标：补齐高频编辑效率能力，并正式开放历史操作。

实现范围：

- [ ] 支持单选、框选、跨小节多选和全选。
- [ ] 支持复制、剪切、粘贴和范围删除。

- [ ] 工具栏和快捷键接入 undo/redo。

验收标准：

### [ ] Iteration 8：保存、导出与发布质量

目标：形成可以稳定保存、恢复和交付使用的 MVP。

实现范围：

- [ ] 实现自动保存 debounce、状态提示、失败重试和离开页面保护。

验收标准：

- [ ] 非法或不支持版本的文档不会污染当前编辑状态。
